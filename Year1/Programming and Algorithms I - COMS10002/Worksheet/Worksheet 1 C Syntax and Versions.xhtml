There are different C versions, the original ANSI C standard, sometimes called C89, then major revisions in 1999 (C99) and 2011 (C11). Read about this history here:

http://en.wikipedia.org/wiki/C_(programming_language)#History

Some features of the C language vary between these different specifications, which means that not all C code is compatible with all C compilers or all other C code. The gcc compiler allows you to pick the version of C you are compiling using the -std= option, e.g. gcc -std=c99 twelve.c -o twelve. Copy the following C code into a file and compile and run it using the C99 standard.

#include <stdio.h>

int main(void) {
// Calculate 3*4 and print the result
    printf("3*4=%d", 3*4);
    return 0;
}

The first line beginning // is a 'comment', which is a request for the compiler to ignore the rest of the line. Re-compile the file with the C89 standard. What happens? Can you find an alternative way of writing a comment which works properly for the C89 standard?
